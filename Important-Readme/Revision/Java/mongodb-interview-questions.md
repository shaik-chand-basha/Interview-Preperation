<h3 dir="ltr" id="what-is-a-collection-in-mongodb?-&lt;span">What is a collection in MongoDB?</h3>
<p dir="ltr" class="p-margin"><span style="background-color: transparent; font-weight: 400; font-style: normal; font-variant: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">A MongoDB collection is a group of documents stored in a single database. It is similar to a table in a relational database, where each document in a collection represents a row in a table. However, the collection doesn’t have a fixed schema, meaning the documents within a collection can be of different data types.&nbsp;</span></p>

<h3 dir="ltr" id="what-is-the-difference-between-find()-and-findone()?-the&lt;c">What is the difference between find() and findOne()?</h3>
<p dir="ltr" class="p-margin">The <code>find()</code> method returns a cursor to multiple documents that match the query criteria. This works by iterating over all the results and fetching the matching documents. On the other hand, the <code>findOne()</code> method returns the first matched document.</p>
<h3 dir="ltr" id="what-is-the-_id-field-in-mongodb?-eachd">What is the _id field in MongoDB?</h3>
<p dir="ltr" class="p-margin">Each document stored in a collection requires a unique identifier. This <code>_id</code> field acts as a primary key to uniquely identify documents in a collection.&nbsp;</p>

<h3 dir="ltr" id="how-does-mongodb-ensure-high-availability?-mongo">How does MongoDB ensure high availability?</h3>
<p dir="ltr" class="p-margin">MongoDB achieves high availability through replication. Replica sets store different copies of data across nodes so that if one node fails, another can take over.</p>

<h3 dir="ltr" id="what-is-a-replica-set-in-mongodb?-arepl">What is a replica set in MongoDB?</h3>
<p dir="ltr" class="p-margin">A replica set in MongoDB is a group of instances that maintain the same dataset. They are deployed in applications that require high availability because if one instance troubles, the system automatically shifts to the next available node in the replica set.&nbsp;</p>
<h3 dir="ltr" id="explain-the-concept-of-aggregation-in-mongodb.-mongo">Explain the concept of aggregation in MongoDB.</h3>
<p dir="ltr" class="p-margin">MongoDB aggregates data from multiple documents and processes it to return a single result. This involves an aggregation pipeline, where documents pass through several stages—each stage’s output becomes the input for the next. A typical pipeline might include stages like match, group, and sort:</p>
<ul style="padding-inline-start: 48px;">
<li>Match: filters documents based on the given criteria.</li>
<li>Group: Performs aggregation operation.</li>
<li>Sort: Sorts the final results the way we need.&nbsp;</li>
</ul>
<h3 dir="ltr" id="what-is-a-capped-collection-in-mongodb?&nbsp;-acapp">What is a capped collection in MongoDB?&nbsp;</h3>
<p dir="ltr" class="p-margin">A capped collection has a fixed size and a limit for the number of documents. When the limit is reached, it automatically overwrites the oldest document and stores the latest information. This concept makes it suitable for use cases like logging and caching.&nbsp;</p>

<h3 dir="ltr" id="does-mongodb-support-acid-transactions?-until">Does MongoDB support ACID transactions?</h3>
<p dir="ltr" class="p-margin">Until version 4.0, MongoDB supported ACID transactions only for single documents. With its multi-document ACID transactions, developers can now ensure <a style="text-decoration: none;" href="https://www.mongodb.com/resources/basics/databases/acid-transactions" target="_blank" rel="noopener">ACID properties</a> across multiple documents within a collection.&nbsp;</p>
<h3 dir="ltr" id="what-is-map-reduce-in-mongodb?-map-r">What is map-reduce in Mongodb?</h3>
<p dir="ltr" class="p-margin">Map-reduce is a data processing paradigm that performs operations on large datasets and outputs aggregated results. MongoDB offers a built-in <code>mapReduce()</code> function consisting of two stages: map and reduce.&nbsp;</p>
<p dir="ltr" class="p-margin">During the map phase, the function processes each document in the collection and generates key-value pairs. These key-value pairs are aggregated in the reduce phase, and operations are performed.&nbsp;</p>
<p dir="ltr" class="p-margin">For example, if you have a collection of text documents, the map function would convert each word into a key and assign a value of 1 to it. The reduce function then sums the values of each key to count the occurrences of each word across the entire collection.</p>
<h3 dir="ltr" id="explain-ttl-indexes-in-mongodb.-gener">Explain TTL indexes in MongoDB.</h3>

<p dir="ltr" class="p-margin">MongoDB provides Time-to-Live (TTL) indexes, which streamline the deletion of expired documents. All you need to do is specify how long a document should be retained, and TTL will automatically remove it once the specified time period has passed.</p>

<h3 dir="ltr" id="what-challenges-might-you-face-while-migrating-from-rdbms-to-mongodb?-&lt;span">What challenges might you face while migrating from RDBMS to MongoDB?</h3>
<p dir="ltr" class="p-margin"><span style="background-color: transparent; font-weight: 400; font-style: normal; font-variant: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Mapping relational database operations to their counterparts in NoSQL can be challenging.</span></p>
<ul style="padding-inline-start: 48px;">
<li><span style="background-color: transparent; font-weight: 400; font-style: normal; font-variant: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">For instance, SQL joins are not straightforward in MongoDB; instead, you need to use the aggregation framework to achieve similar functionality.</span></li>
<li><span style="background-color: transparent; font-weight: 400; font-style: normal; font-variant: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Another challenge is the data stored in structured tables will need transformation to fit into MongoDB’s BSON storage format.&nbsp;</span></li>
<li><span style="background-color: transparent; font-weight: 400; font-style: normal; font-variant: normal; text-decoration: none; vertical-align: baseline; white-space: pre-wrap;">Additionally, while RDBMS provides robust ACID compliance, MongoDB only offers document-level ACID properties, which means complex ACID transactions may require extra attention.</span></li>
</ul>
