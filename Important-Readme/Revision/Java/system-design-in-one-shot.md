(RPS)</li><li>Transactions per second (TPS)</li></ul><p>A system can have:</p><ul><li>Low latency but low throughput</li><li>High throughput but high latency</li></ul><p>The ideal system max-sm:pointer-events-none"></p><h2 id="the-three-properties-explained" class="relative group scroll-mt-6">The Three Properties Explained<a aria-label="jump-to-section-links" class="pl-2 font-bold href="#availability">#</a></h3><p>Availability means:</p><ul><li>Every request receives a response</li><li>The system continues to function even if some nodes fail</li></ul><p>The response does not need to be correct or latest, but the system should not refuse requests.</p><p>Example:</p><ul><li>One database replica crashes</li><li>Other replicas continue serving traffic</li></ul><hr><h3 id="partition-tolerance" class="relative group scroll-mt-6">Partition Tolerance<a aria-label="jump-to-section-links" class="pl-2 font-bold no-underline opacity-0 hover:underline text-primary Because of this, <strong>partition tolerance is mandatory</strong> in distributed systems.</p><hr><h2 id="the-core-idea-of-cap-theorem" class="relative group scroll-mt-6">The Core Idea of CAP (Availability + Partition Tolerance)</li></ul><p>Impossible combination:</p><ul><li>CAP (all three at once)</li></ul><hr><h2 id="why-can-we-not-have-all-three" class="relative group scroll-mt-6">Why scroll-mt-6">If we choose Consistency:<a aria-label="jump-to-section-links" class="pl-2 font-bold no-underline opacity-0 hover:underline text-primary group-hover:opacity-100" href="#if-we-choose-consistency">#</a></h3><ul><li>Stop serving requests until network is restored</li><li>Prevent inconsistent writes</li></ul><p>Result:</p><ul><li>Consistency trading platforms</li><li>Account balances</li></ul><hr><h3 id="choose-ap-when" class="relative group scroll-mt-6">Choose AP When:<a aria-label="jump-to-section-links" class="pl-2 font-bold id="why-do-we-need-event-driven-architecture" class="relative group scroll-mt-6">Why Do We Need Event-Driven Architecture?<a aria-label="jump-to-section-links" class="pl-2 font-bold no-underline }</p><p>Consumers fetch required data separately.</p><p>Pros:</p><ul><li>Lightweight events</li></ul><p>Cons:</p><ul><li>Extra database calls</li></ul><hr><h3 id="event-carried-state-transfer" href="#event-carried-state-transfer">#</a></h3><p>Event carries full data.</p><p>Example: order_created { order_id, user_id, amount, items }</p><p>Pros:</p><ul><li>No extra fetch
