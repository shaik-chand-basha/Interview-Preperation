These are pre-defined sets of libraries bundled together for common tasks.</p><p>For example:</p><ul><li><code class="bg-gray-200 dark:bg-gray-700 rounded-sm p-[2px] break-all">spring-boot-starter-web</code> includes everything needed to build a REST API (Spring MVC, Jackson, embedded Tomcat, etc.).</li><li><code class="bg-gray-200 dark:bg-gray-700 rounded-sm p-[2px] break-all">spring-boot-starter-data-jpa</code> brings in Hibernate, JPA API, and a database driver (like H2 for development).</li></ul><p>Also, Spring Boot uses a <strong>parent POM</strong> (in Maven) to manage compatible versions of libraries, so you don’t have to worry about version conflicts.</p><h3 id="8-what-is-the-purpose-of-the-applicationproperties-or-applicationyml-file" class="relative group scroll-mt-6"><strong>8.
